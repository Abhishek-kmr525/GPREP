# Project Operations

## Target timeline

- Target completion date: `2026-03-25`
- Working mode: weekly sprint execution
- Planning driver: MOM from weekly sprint calls

## What Codex will manage each cycle

- convert MOM into action items
- update implementation plan
- maintain documentation notes
- prepare Trello-ready card content
- prepare GitHub-ready branch/commit/PR notes
- track blockers, dependencies, and client decisions
- update content/action status based on latest client inputs

## Weekly operating flow

1. You share MOM after sprint call.
2. I extract:
   - decisions
   - action items
   - blockers
   - dependencies
   - content requests
   - design or scope changes
3. I update:
   - sprint tracker
   - Trello brief/cards
   - documentation log
   - GitHub execution notes
4. We execute development work for the sprint.
5. I prepare the next sprint summary and open items.

## Inputs I need from you each week

- MOM or rough notes
- any new design/source files
- content files from client
- change requests from client
- repo/branch status if code already exists elsewhere

## Constraints to keep in mind

- timeline is short
- content dependencies are likely the biggest risk
- emergency-mode requirements are high priority
- Drupal remains the implementation direction
