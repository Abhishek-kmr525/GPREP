# Post-Launch Support Notes

## Immediate Post-Launch Monitoring

- check homepage, News, FAQ, Resources, Contact, Map, and emergency mode behavior
- review editor login and role access
- review contact form submissions
- monitor any broken links or missing media

## Editorial Support Areas

- News publishing and incident tagging
- emergency mode usage
- homepage content updates
- menu and footer maintenance
- Webform field updates and submission exports

## Technical Support Areas

- Drupal cache rebuilds after major content or deployment changes
- module and theme update review
- server health, PHP, and database checks
- GitHub-to-server sync discipline

## Recommended Support Cadence

### Weekly
- review Trello status
- review client change requests
- verify server and GitHub remain aligned

### After Every Sprint Or Client Call
- convert MOM decisions into action items
- update documentation if any workflow changed
- update Trello due dates and priorities

### Before Any Emergency Activation
- confirm the latest News item is published
- confirm the ticker is current
- confirm the emergency homepage on desktop and mobile

## Escalation Guidance

- content issue: assign to content editor or site manager based on scope
- emergency settings issue: site manager only
- deployment or server issue: engineering
- permissions issue: site manager plus engineering
